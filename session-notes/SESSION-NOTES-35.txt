HEADLINE

Two architectural directives from Bill anchored the afternoon's work:

1. "WHY NOT JUST USE THE LABELS WE GAVE IT"
   The hero subtitle was rendering "Hunter Hunter Derby · 2 rounds, 2
   judges" because the code prefixed "Hunter " on top of a label that
   already started with "Hunter". Bill: "we gotta work on your human
   speak". Rule: code uses canonical labels as-is, never re-prefixes.
   Page context (we're already on a horse-show page) does the rest of
   the work.

2. "MAKE SURE THAT LOGIC TEST IS IN THERE … LOTS THIS WILL BE USED FOR"
   The derby-type override needed gating on class_mode === 2. Bill
   pushed for that to be a NAMED gate function — derbyTypeLabel(cls) —
   so the same pattern applies to judges grid, derby components,
   combined-total column, forced-placings layout, etc. Six gates now
   live in west-format.js. Future surfaces (judges grid render, stats
   page, live ticker) just call the gate; the policy can't be skipped.

PART H — classDescription(cls) — HUMAN ENGLISH HERO LINE
═══════════════════════════════════════════════════════════════════════════

Bill: "instead of telling us the useless class mode 2 can we put
what we know about the class in human english". Then later: "we gotta
work on your human speak ... why not just use labels we gave it".

WEST.format.classDescription(cls) synthesizes mode + scoring type +
round count + judges + derby variant + equitation flag into one line.

Lookup tables:
  HUNTER_MODES = { 0: 'Over Fences', 1: 'Flat',
                   2: 'Hunter Derby', 3: 'Special' }
  HUNTER_SCORING_TYPES = { 0: 'Forced placings',
                           1: 'Scored',
                           2: 'Hi-Lo (drop high + low judges)' }
  DERBY_TYPES (zero-indexed — see DECISION below)

Examples:
  Class 1001 (Mode 2, derby_type=0): "International Derby · 2 rounds, 2 judges"
  Class 800  (Mode 0, 7 judges):     "Over Fences · 7 judges"
  Class 901  (Mode 0, Forced):       "Over Fences · Forced placings"
  Class 902  (Mode 0, Forced + Eq):  "Equitation · Forced placings"
  Class 925  (Mode 3, 3R, 2J):       "Special · 3 rounds, 2 judges"
  Jumper 23  (Method 13):            "II.2b · Immediate JO"  (unchanged)

Rules:
  - Use canonical labels as-is — never re-prefix "Hunter " onto
    "Hunter Derby"
  - "Scored" is implicit (skipped); Forced and Hi-Lo called out
  - Counts only mentioned when > 1
  - is_equitation overrides headline to "Equitation"
  - derby variant overrides "Hunter Derby" with the specific name (only
    when class_mode === 2)
  - Sponsor and trophy stay separate hero subtitle lines (not part of
    classDescription).

PART J — NAMED GATES PATTERN (architectural lock-in)
═══════════════════════════════════════════════════════════════════════════

Bill: "make sure that logic test is in there ... lots this will be
used for". Triggered by the derby-type-only-when-class_mode-2 rule;
expanded into a general policy.

Pattern: every conditional render decision gets a NAMED gate function
that bakes in EVERY relevant condition. Callers get a single yes/no.
No `if (cls.class_mode === 2 && cls.derby_type != null && ...)` chains
scattered across templates.

GATES NOW IN west-format.js:

  derbyTypeLabel(cls)     → variant name only when class_mode === 2
                              and derby_type is in map; null otherwise
  judgesGridApplies(cls)  → true if num_judges > 1 AND scoring_type !== 0
  combinedTotalApplies(cls) → true if num_rounds > 1 AND scoring_type !== 0
  derbyComponentsApply(cls) → true if class_mode === 2 AND scoring_type !== 0
  forcedPlacings(cls)     → true for hunter scoring_type=0 OR jumper
                              method=7 modifier=0 (cross-lens)
  riderPrimary(cls)       → true if is_equitation === 1
  championshipMarker(place, isChampionship) → "Ch"/"Res"/"" (place gate)

ARCHITECTURAL DECISION 19 (added below): gate functions are the only
public API for "should I render X" — direct field reads in template
code are a smell; refactor into a gate.

When the JUDGES GRID render lands (future phase), the template just
calls WEST.format.judgesGridApplies(cls). Gate is already there;
policy can never be skipped.

ARCHITECTURAL DECISIONS LOCKED THIS SESSION (continued from session 34)
═══════════════════════════════════════════════════════════════════════════

19. NAMED GATES FOR CONDITIONAL RENDERING.
    Every "should I show X?" decision lives in a named gate function
    in west-format.js. Templates call the gate and get yes/no — they
    never reach into class fields directly to compose a condition.
    See Part J for the seven gates currently in place; pattern extends
    to future flags (championship variants, derby components, judges
    grid, etc.).

20. CANONICAL LABELS, NEVER RE-PREFIX.
    Lookup tables (HUNTER_MODES, DERBY_TYPES, JUMPER_METHODS,
    HUNTER_SCORING_TYPES) hold the human-facing strings. Code does not
    decorate them with "Hunter " or other prefixes — page context is
    enough. Adding a prefix is the bug that produced "Hunter Hunter
    Derby" before this session.

21. DERBY_TYPES IS ZERO-INDEXED.
    Ryegate dropdown writes 0 for the first option (International).
    Spec doc HUNTER-METHODS-REFERENCE.md was 1-indexed and is wrong;
    code is the source of truth. Verified empirically by Bill setting
    the dropdown to "National" and reading the header byte as 1.

22. DNS-LIKE ENTRIES ARE FILTERED FROM PUBLIC RESULTS.
    isDnsLike(entry) — cross-lens. No place + no round data + no status
    + no combined total → drop the row entirely. Operator still sees
    them in admin; public viewers don't need to see entries that never
    rode.

23. RESULTS LAYOUT IS A PER-SHOW ADMIN SETTING.
    shows.results_layout (migration 016): 'stacked' (default) or
    'inline'. Operator picks via admin show dialog; public class.html
    honors. No per-page toggle — single point of control.

24. HUNTER ROUNDS STACK CHRONOLOGICALLY.
    Stacked layout for hunters renders Round 1 on top, Round 2 below
    (no reversal). Hunter rounds are independently scored, not
    cumulative — there's no climactic round to put on top.
    Jumper still reverses (Jump Off above Round 1) per its convention.

25. PUBLIC SURFACES NEVER BYPASS flagFor.
    (Restated from session 34.) Country flag visibility is gated by
    operator's Ryegate ShowFlags toggle. Code-level enforcement via
    WEST.format.flagFor(cls, entry). flag(code) is a primitive that
    bypasses; the policy gate is flagFor.
