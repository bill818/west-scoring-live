1. CLASS FINALIZE DETECTION — EXHAUSTIVE INVESTIGATION (Day 1)
   Spent full day investigating every possible file/network signal for
   detecting when operator presses Upload Results / Finalize & Close.

   Dead ends documented (don't re-explore):
   - netstat TCP polling (68.178.203.100) — blind to HTTP keepalive
   - typeperf per-process IO counters — Upload click blends into baseline
   - log.txt scraping — error-only, not touched during uploads
   - master.dat — entry list, not upload registry (false alarm)
   - config.dat — changed from config activation, not upload
   - Passive pcap from LAN — switched network, needs drivers

   Working signals found:
   - .tod file NEW = first-time class finalize (jumper, AlgeTimy/FDS)
   - .tod file size delta = subsequent finalize with new data
   - .cls size growth = score save (already in watcher)
   - Hunter: Export/{N}.csv + tsked.csv S flag (existing)

   Incidental discoveries:
   - Ryegate uploads via HTTPS 443 (not FTP 21) to 68.178.203.100
   - FDS field in config.dat = timer brand, not FTP label
   - Old ryegate.com endpoint is LIVE production — don't decommission
   - No background heartbeat — "periodic" was operator's click cadence

2. RYEGATE.LIVE PEEK DETECTION (Day 2)
   Discovered ryegate.live page structure for detecting class state:
   - "Please Check Back Later" = NOT_STARTED
   - "ON COURSE" + "PREVIOUS EXHIBITOR" = LIVE
   - "X of Y Competed" = IN_PROGRESS
   - "Order of Go" = ORDER_POSTED (new state)
   - Bare results table (no indicators) = UPLOADED

   Built peekClass() with randomized 15-30s polling interval.
   Goes dormant on UNKNOWN/ERROR or test paths (NONWEST).
   ORDER_POSTED event forwarded to Worker for OOG badge.

3. 5-LAYER FINALIZE DETECTION HIERARCHY
