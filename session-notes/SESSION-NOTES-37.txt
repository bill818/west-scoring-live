PART A — PAGES PREVIEW PLUMBING
═══════════════════════════════════════════════════════════════════════════

WRANGLER CONFIG SPLIT
  - The repo had ONE wrangler.toml — the Worker config (name=
    west-worker, [[d1_databases]] etc.). Pages would 404-warn on every
    deploy because the file was missing pages_build_output_dir.
  - Tried adding a separate wrangler.pages.toml with
    `wrangler pages deploy --config wrangler.pages.toml` — Cloudflare
    rejects --config for Pages ("Pages does not support custom paths
    for the Wrangler configuration file").
  - Resolution: rename the Worker file to wrangler.worker.toml and let
    the new pages config take the canonical name.
      wrangler deploy           --config wrangler.worker.toml   (Worker)
      wrangler pages deploy ... (no --config — picks up wrangler.toml)
  - Updated deploy.bat to pass --config wrangler.worker.toml.
  - Updated worker_deploy_setup.md memory to document the split.

MAC SHELL DEPLOY SCRIPT
  - Bill thought I'd already done this; I'd only updated the Windows
    .bat. Created deploy-preview.sh as the Mac counterpart so future
    deploys from this Mac are a single ./deploy-preview.sh command —
    sources ~/.cloudflare-env, sources nvm, stages v2 root + v3/pages
    + v3/js + assets, deploys, cleans up.
  - chmod +x'd. Tested end-to-end. From this commit onward I use it
    (not the manual rm-mkdir-cp-wrangler shell sequence).

DEPLOYMENT TARGET
  - URL: preview.westscoring.pages.dev/v3/pages/{index,show,ring,class,admin}.html
  - Cloudflare Pages auto-redirects .html → no-extension (308) which
    is fine; the redirected URL serves the static file.
  - Browser cache surprised us — Bill needed Firefox (which had a
    fresh cache) to actually see CSS changes after a Chrome session
    holding old assets. Documented in the workflow.

PART B — TYPOGRAPHY OVERHAUL
═══════════════════════════════════════════════════════════════════════════

THREE PAIRINGS TRIED (Bill iterating per-on-device-look):
  v1: Playfair Display (serif) + Source Sans 3   → Bill: "no I don't like the serif font"
  v2: Outfit (geometric sans)  + Source Sans 3   → Bill: "better but still no"
  v3: Oswald (condensed sport) + Lato            → "yes that is right"

Final state:
  Headers (logo, hero name, show/section/empty-state titles, ring card
  names, dialog headers, active-num):
    'Oswald', sans-serif — weight 700 (Oswald maxes at 700; old 900
    rules dropped to 700 to avoid synthesized-bold ugliness).
  Body:
    'Lato', sans-serif — weights 300/400/700 + 400 italic. Source Sans
    3 rules removed. Lato lacks 500/600 → existing font-weight:500
    falls back to 400; font-weight:600 falls back to 700. Acceptable
    for current usage; no need to remap.
  Mono / labels:
    'DM Mono' — unchanged (column headers, DT badges, breadcrumbs,
    metadata lines).

iOS ROTATION FIX
  Bill: "i turn my phone side ways and the font gets disproportionately
  bigger". Default iOS Safari behavior — auto-text-inflate based on
  viewport width. Killed via:
    html{-webkit-text-size-adjust:100%; text-size-adjust:100%;}
  on the global resets.

PART C — IDENTITY COLUMN HEADER REDESIGN
═══════════════════════════════════════════════════════════════════════════

Bill: the standings header just says "Horse / Rider" — should say
  Horse-Rider
  Owner
  Breeding
... and equitation should swap line 1 to Rider-Horse.

IMPLEMENTATION
  - Header definitions now use a new h.html field on column descriptors
    (in addition to the existing h.label). The th renderer in 4 places
    (jumper inline + stacked, hunter inline + stacked) checks
    `h.html != null ? h.html : escapeHtml(h.label)`. Allows raw HTML
    spans for the identity column without affecting other columns.
  - Identity header HTML: three <span class="hd-line"> blocks for
    Horse-Rider / Owner / Breeding. Equitation flips line 1 to
    Rider-Horse. Other modes share one rule via the existing
    riderPrimary flag in hunter buildForcedColumns / jumper builders.

CSS
  .results-table thead .hd-line{display:block;line-height:1.35;}
  Generic, scoped only to thead so cell content keeps wrapping
  normally.

OWNER ITALICS, BREEDING IN PARENS
  - .entry-owner now italic (DM Mono italic, 11px, muted).
  - .entry-pedigree wrapped in parens via ::before/::after — keeps
    data layer clean, no string concatenation in templates:
        .results-table .entry-pedigree::before{content:'(';}
        .results-table .entry-pedigree::after{content:')';}

PHONE BREAKPOINT (≤ 480px)
  Bill: "drop breeding when it gets down to a phone screen"
  - .results-table thead .entry-horse-rider .hd-breeding{display:none;}
  - .results-table .entry-pedigree{display:none;}
  Header and cell hide together — never out of sync.

31. WRANGLER CONFIG SPLIT (worker vs pages).
    Pages doesn't accept --config <path>, so Pages must own the
    canonical wrangler.toml. The Worker uses wrangler.worker.toml and
    is invoked as `wrangler deploy --config wrangler.worker.toml`.
    deploy.bat / deploy-preview.{bat,sh} updated; memory updated.

32. THREE-FONT STACK FOR PUBLIC SURFACES.
    Headers Oswald 500/600/700, body Lato 300/400/700+italic, labels
    DM Mono 400/500. Loaded via shared Google Fonts URL on every v3
    page. Sport/results vibe with editorial body legibility.
    iOS auto-text-inflate killed at the html level — non-negotiable.

33. HEADER COLUMN DESCRIPTORS SUPPORT RAW HTML (h.html).
    Th renderers across both lens templates check `h.html != null ? h.html
    : escapeHtml(h.label)`. Default path stays escape-safe; raw-HTML
    columns (identity 3-line stack, future TA second line, future
    composite headers) opt in explicitly. Pattern documented in the
    new feedback memory.

34. PHONE BREAKPOINT IS 480px (TYPE-SAFE OVER OTHER WIDTHS).
    First width-driven hide rule was added for breeding (header + cell
    in lockstep). Future "hide on phone" rules should reuse 480px so
    the responsive cliff is consistent.

35. PER-CLASS LENS TINT VIA CSS-CLASS-ON-HERO.
    Single-element wrapper class (hero + hero-{h|j|t|u}) drives the
    background gradient. Pattern is reusable: any future surface that
    wants lens-aware visual treatment can mount the same class names
    and add tinted CSS rules.

36. WRANGLER CONFIG / MAC DEPLOY SCRIPT IS REPO-CHECKED-IN.
    deploy-preview.sh joins deploy-preview.bat at the repo root. Both
    must stay in sync when staging changes (e.g. when a future
    surface adds a new directory under v3/).
