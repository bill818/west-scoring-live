2. False class-focus fix (Bill's "Class Opened" theory — CONFIRMED)
   Root cause: /v3/postCls → DO /scores-update ran a `cls_lock` block
   that set is_live=true on ANY class's first .cls write with scores,
   bypassing the B+intro pair gate. Ryegate writes the .cls on every
   score post and grabs class context early → false openings + focus
   theft before the operator's INTRO+31000 pair.

   Fix (west-worker.js, all server-side — NO engine change):
   • New `opened_at` field — canonical "Class Opened" timestamp. Set
     ONLY by the B+intro pair in _processLiveTriggers (port 31000
     Channel B + INTRO frame fr=1/11 within LIVE_PAIR_WINDOW_MS) and by
     the flat-class .cls path. Sticky; mirrors finalized_at.
   • cls_lock gated to FLAT classes only (hunter, class_mode===1) — the
     one legit no-B-pair workflow Bill flagged. /v3/postCls now sends
     class_type+class_mode in the /scores-update body so the DO can tell.
     Non-flat .cls writes still STORE scores (standings pre-warm) but do
     NOT open/surface/focus the class.
   • Self-heal: a non-flat entry still live via the OLD cls_lock with no
     opened_at gets is_live=false (unlive_reason='false_cls_lock_
     corrected') on its next .cls write.
   • classes[] filter tightened: was `is_live || live_since!=null ||
     is_final`; now `opened_at!=null || is_final`. Backfill loop adopts
     live_since→opened_at for legit intro+focus / flat_cls triggers so
     the deploy didn't drop any genuinely-live class mid-show. cls_lock
     entries are deliberately NOT backfilled → they drop immediately.
   • _focusedClassId pair-loop keys off opened_at instead of is_live.

   NOT changed (deliberately, mid-show risk): the _focusedClassId eager
   fallbacks (last_focus/last_scoring/class_meta). They only fire when
   NO class is opened (cold-start render) and can't surface a class
   (classes[] now needs opened_at). Revisit only if focus-without-
   surface still misbehaves.
