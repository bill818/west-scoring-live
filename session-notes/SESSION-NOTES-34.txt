   Round labels until the framework is settled.

   computer." Operator's H[26] ShowFlags toggle is THE override. Code
   enforces this via WEST.format.flagFor(cls, entry); a primitive
   flag(code) exists but is documented "use flagFor instead."

PART 2 — TIMEZONE PER SHOW
═══════════════════════════════════════════════════════════════════════════

Bill flagged that the show "ends at 11:59pm of the last day" — and that
should be in the show's local timezone, not UTC. Test show was Devon PA
which is Eastern time; the existing UTC-driven date math was off by up
to 5 hours.

Decision: explicit per-show timezone field, not derived from the
location text. "Devon, PA" → "America/New_York" parsing is fragile;
operator picks from a fixed list in admin.

MIGRATION 013 — shows_timezone
  ALTER TABLE shows ADD COLUMN timezone TEXT DEFAULT 'America/New_York'
  Backfill is automatic via the DEFAULT — existing test show got
  America/New_York set on the migration commit.

WORKER (/v3/createShow + /v3/updateShow)
  Both endpoints now accept `timezone` field. createShow defaults to
  America/New_York if missing/empty. updateShow rejects empty string
  but allows the field to be omitted.

ADMIN (v3/pages/admin.html)
  Show dialog gains a Timezone select with 10 options:
    Eastern, Central, Mountain, Phoenix (no DST), Pacific, Alaska,
    Hawaii, London, Paris, Sydney
  Form-hint: "Drives when the show flips to Past on the public index
  (11:59pm local)."

PUBLIC PAGES (index.html, show.html)
  Both compute "today" in the show's local timezone:
    new Date().toLocaleDateString('en-CA', { timeZone: tz })
  en-CA gives YYYY-MM-DD format. Compared to show.end_date directly.

Pulled all duplicated page-glue out of the four public pages and the

NEW: [v3/js/west-api.js]
  WEST.api.AUTH = 'west-scoring-2026'
  WEST.api.BASE = 'https://west-worker.bill-acb.workers.dev'
  WEST.api.fetchJson(path)  → JSON, throws on !ok
  WEST.api.queryParam(name) → URL search param helper
  Why: single place to change worker URL or rotate auth.

EXTENDED: [v3/js/west-format.js]
  Added: escapeHtml, time(sec), faults(n), ordinal(n),
         dayLabel(iso), dayLabelLong(iso),
         flag(code) — primitive,
         flagFor(cls, entry) — POLICY GATE (the important one)
  FLAGS map (200 codes) lives here; templates module consumes via
  WEST.format.flag.

POLICY GATE — flagFor(cls, entry)
  Returns the rendered flag string ONLY when cls.show_flags is
  truthy AND entry.country_code is non-empty. Code-level enforcement
  of Bill's "scoring computer is the only override" directive.
  All current and future surfaces (results, live, stats, display,
  any new layout) MUST route through flagFor — they CAN'T accidentally
  bypass the operator's Ryegate ShowFlags toggle by reaching for
  flag(code) directly. Documented as such in the JS-MODULES doc.

PAGES STRIPPED OF DUPLICATION
  show.html, ring.html, class.html each lost their inline copies of:
    - escapeHtml
    - fetchJson + AUTH/BASE constants
    - getQueryParam
    - fmtDayLabel
  Each now has a 4-line preamble:
    const escapeHtml    = WEST.format.escapeHtml;
    const fetchJson     = WEST.api.fetchJson;
    const getQueryParam = WEST.api.queryParam;
    const fmtDayLabel   = WEST.format.dayLabel;  // or dayLabelLong
  Page bodies stay page-specific (breadcrumbs, hero composition,
  fetch orchestration). No more diverging primitives.

TEMPLATES MODULE FOLLOW-UP
  west-jumper-templates removed its private FLAGS map — uses
  WEST.format.flagFor inline. escapeHtml is a thin alias to
  WEST.format.escapeHtml. The module's surface is purely
  jumper-template logic now; no general utility duplication.

13. SHOW LIFECYCLE:
    built → status=pending (UI: "upcoming")
    heartbeat → status=active (UI: "active")
    end_date < today (in show timezone) → UI: "past"
                       regardless of stored status
    Status field is authoritative within the date window. End_date is
    a hard ceiling — past it, show goes to past group always.
    (Adds to SESSION-32/33 decisions 1-12.)

14. SHOW TIMEZONE IS EXPLICIT.
    Per-show field, not derived from location text. Default
    America/New_York. Operator picks from a fixed admin list. All
    "today"/"past"/"active" comparisons compute today in the show's
    local TZ. Avoids 5-hour UTC drift on East Coast shows.

15. JUMPER LENS USES TEMPLATES MODULE; HUNTER LENS WILL MIRROR.
    west-jumper-templates.js owns: detection (method → template id),
    1R/2R/3R/EQ/TEAM renderers, jumper-specific helpers (round cell,
    flag span, place check via west-rules). When hunter results
    land: parallel west-hunter-templates.js with the equivalent
    surface for hunter modes. The two never reach into each other.

16. SCORING-COMPUTER IS THE SINGLE SOURCE OF TRUTH FOR FLAG VISIBILITY.
    Operator's Ryegate H[26] ShowFlags toggle is THE authority.
    Code-level enforcement: every public surface routes through
    WEST.format.flagFor(cls, entry). No per-page "always show"
    bypass exists. Unchecking ShowFlags in Ryegate erases flags
    everywhere on next /v3/postCls.

17. ROUND LABELS STAY GENERIC R1/R2/R3 — FOR NOW.
    Phase 1/Phase 2 (Method 9), Jump-Off (Methods 2/13), Winning Round
    (Method 15) etc. labeling layers in later. Pinned to keep the
    template framework simple while the rest of the public surface
    is being built out. Bill: "i cant keep of two manny thing in my
    head at once."

18. CENTRAL-BY-LENS ARCHITECTURE.
    Cross-lens primitives (status, format, rules) shared. Lens-
    specific concerns (jumper templates / hunter templates / lens
    parsers) split into per-lens modules. Templates within a lens
    are parameterized by detection (method) but live in one module

      Cross-lens primitives: west-status, west-format, west-rules
      Lens-specific:         west-{jumper,hunter}-templates,
