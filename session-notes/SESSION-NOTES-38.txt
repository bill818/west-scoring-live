MIGRATION 019 — classes_time_allowed
  ALTER TABLE classes ADD COLUMN r1_time_allowed REAL;
  ALTER TABLE classes ADD COLUMN r2_time_allowed REAL;
  ALTER TABLE classes ADD COLUMN r3_time_allowed REAL;

  Jumper-only fields. Hunter classes always NULL — TA isn't a hunter
  concept. Class-level metadata (not per-entry) so every public surface
  reads the same value.

WORKER — parseClsHeaderV3 J/T branch
  Added taOf() helper (parseFloat → null when 0/blank/NaN — 0-second
  TA never displays as a real value):
    r1_time_allowed: taOf(cols[8])
    r2_time_allowed: taOf(cols[11])
    r3_time_allowed: taOf(cols[14])
  Column indexes match v2 watcher (west-watcher.js:745-751). Hunter
  branch unchanged → returns object without these fields → UPSERT
  binds nullish-coalesce to null.

WORKER — /v3/postCls UPSERT
  Added the three columns to:
    - INSERT column list (between ribbons_only and parse_status)
    - VALUES placeholder count (28 → 31 ?'s)
    - ON CONFLICT update list
    - bind() params (parsed.r1_time_allowed ?? null, etc.)
  /v3/listClasses uses SELECT c.* so the new columns flow through the
  API automatically — no API code change.

═══════════════════════════════════════════════════════════════════════════
PART B — BACKFILL ENDPOINT
═══════════════════════════════════════════════════════════════════════════

POST /v3/reparseClassHeaders
  Body: { slug?: 'show-slug' }   (omit slug → all classes, all shows)

  Walks classes (optionally filtered by show slug), pulls each class's
  archived .cls bytes from R2 at the stored r2_key, runs
  parseClsHeaderV3, and UPDATEs the columns this migration adds.
  Idempotent. Skips classes with NULL r2_key (shouldn't happen) or
  missing R2 objects.

  Used today to backfill all 71 existing classes — single curl, single
  pass, all the new fields populated where applicable. Future
  migrations that add column-on-classes can reuse the endpoint and
  swap the UPDATE list to populate their columns. Lives at the same
  spot as /v3/recomputeJudgeRanks (admin-only safety-valve pattern).

  Auth: requires X-West-Key (same as every other v3 admin endpoint).

  Returns: { ok: true, scanned: N, updated: N, skipped: N, errors: N }

  Backfill verified post-run:
    - 47/51 T classes have r1_time_allowed (4 classes had blank R1
      TA in the source .cls — correct per data, not a parse bug)
    - 51/51 T classes have r2_time_allowed (default 30s often present
      in Ryegate even on 1R classes — harmless because the formatter
      gates on method-aware "is multi-round")
    - 16/16 H classes correctly NULL
    - 4/4 U classes correctly NULL

═══════════════════════════════════════════════════════════════════════════
PART C — FORMAT (cross-lens primitive)
═══════════════════════════════════════════════════════════════════════════

WEST.format.timeAllowedSummary(cls) — method-aware one-liner.
Returns:
   1R / Speed / Optimum  →  "TA 30s"
   Two-Phase (m=9)       →  "PH1 30s · PH2 31s"
   2R + JO (m=2/etc)     →  "R1 65s · JO 30s"
   3R + JO (m=3/14)      →  "R1 65s · R2 70s · JO 30s"
   hunter / no TA        →  "" (empty — caller skips render)

Rounds that don't actually have a TA are silently skipped (not
rendered as "0s" or "—"). Method 6 with modifier=1 promotes to 2R per
the same override that roundLabel uses — single source of truth for
"is this a multi-round method".

Lives in west-format.js (cross-lens primitives module). Consistent
with classDescription / methodLabel / dayLabel placement — formatters
that take a class and return display strings.

ARCHITECTURAL DECISIONS LOCKED THIS SESSION
═══════════════════════════════════════════════════════════════════════════

37. TIME-ALLOWED IS CLASS-LEVEL METADATA, NOT PER-ENTRY.
    Stored on `classes` (one value per class per round, three columns).
    Same data feeds every surface — hero today, future live page,
    future per-round timing display, future stats correlations.
    Hunter classes leave the columns NULL.

38. METHOD-AWARE TA SUMMARY LIVES IN west-format.
    timeAllowedSummary(cls) — single source of truth for the human-
    facing TA string. Centralized so the live page, admin tooltips,
    stats summaries, etc. all render the same wording. Uses the same
    method-axis logic as roundLabel (method 9 → PH1/PH2; method 6+mod1
    → 2R; methods 3/14 → 3R).

39. /v3/reparseClassHeaders IS THE BACKFILL POSTERN FOR ANY classes-
    LEVEL MIGRATION.
    When a future migration adds new columns to `classes` that
    parseClsHeaderV3 should populate, the recipe is:
      1. Migration adds columns
      2. Update parseClsHeaderV3 to return them
      3. Update postCls UPSERT to bind them
      4. Update reparseClassHeaders' UPDATE statement to write them
      5. Apply migration, deploy worker, curl reparseClassHeaders
    Steps 1-3 are the existing pattern. Step 4 is the new affordance
    this session adds — no longer need a one-off backfill script.

40. TA RENDERS IN THE HERO, NOT PER-ROUND-COLUMN.
    Bill rejected per-row TA after seeing it. The hero subtitle is the
    single sanctioned surface for TA. Future surfaces (live ticker,
    display.html) should reuse the same formatter for consistency.
