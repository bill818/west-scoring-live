SHOW LOCK / END-DATE
- Root-caused a chronic "show is locked, worker rejected" loop:
  hits-culpeper end_date was yesterday, autoCompleteShow flipped
  status=complete on every watcher post, instantly undoing any
  admin "Set Active" click.
- Fix: /admin/updateShow now auto-bumps end_date to today when
  admin sets status=active with a stale end_date. Manual patch
  bumped end_date to 2026-04-20 for the current show.

U→T INFERENCE OVERRIDE
- Worker was preferring the raw .cls header's "U" type over the
  watcher's inferred type, so class 809 (II.2a) kept stats-page-
  ing as "U / Not a Jumper Class" even after the watcher flipped
  U→T. Fixed to prefer body.classType when non-U.
- Cascade bug: U-branch in computeClassResults didn't filter by
  hasGone, so pending entries polluted standings. Added a
  defensive filter.

II.2A JO-N OVERLAY (method 2 only)

  "JO-1, JO-2..." in R1 ride order instead of a bunch of tied
  "1"s. Once any clear gets JO data, overlay drops out and
  Ryegate's place flows through for every row.
- Scoped to method 2. II.2b (13) excluded because its JO is
  immediate — places populate progressively, no pre-JO gap.
- Worker now ships rideOrder on computed entries so the sort
  tiebreaker and overlay can reason about return order.

WD / DECLINED-JO
- Ryegate doesn't write text status (col[82]/col[83]) for a
  rider who WDs from JO without riding. It DOES write the
  numeric code (col[21] R1, col[28] R2) — confirmed 3=HF, 4=WD.
- Watcher now has a numeric → text fallback so the .cls is
  self-sufficient when the UDP finish frame is missed.
- Worker overlayFinishStatus: on UDP FINISH events with a
  status code, injects r{round}StatusCode into classData KV,
  computed KV, AND D1 results row (so it survives KV expiry).
- Sort on stats JO standings now uses Ryegate's place first —
  a WD that was the only clear still shows 1st, not dumped to
  the bottom.
