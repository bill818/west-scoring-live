FARMTEK TWO-PHASE TRANSITION (from Bill's description)
------------------------------------------------------
Farmtek II.2d (method 9) two-phase pattern:
  Phase 1: clock runs with TA:91 (or whatever)
  Phase 1 finish: decimal time appears (FINISH)
  Operator presses transition button: "RANK: " (empty)
  ~5 second silence: NO UDP frames
  Phase 2: UDP resumes at elapsed=5 (NOT 0!), TA swaps to
           Phase 2 value (e.g. 59), clock continues 6,7,8...

Key detection signals:
  - Same entry (horse hasn't changed)
  - elapsed drops dramatically (86→5)
  - TA changes (91→59)
  - Clock never resets to 0 — comes back wherever it is

Watcher handles two paths:
  1. FINISH→ONCOURSE (lastPhase='FINISH'): line 2046,
     RIDE_START fires with inferRound detecting TA change
  2. ONCOURSE→ONCOURSE TA change (FINISH missed): line 2063,
     re-posts with new round info

Post-FINISH lock (finishLockUntil) prevents buzzer noise from
creating false ONCOURSE events. The lock allows legitimate phase
transitions through because they have a TA change.

No real two-phase UDP logs captured yet — morning classes ran
on the pre-restart watcher. Next II.2d class will be captured
in the UDP + peek logs with prominent PHASE TRANSITION logging.
