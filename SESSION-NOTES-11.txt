## UDP PROTOCOL — FULLY CONFIRMED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Scoreboard port: config.dat col[1] (e.g. 29711)
Live data port:  scoreboardPort - 496 (e.g. 29215) — LOCKED by Ryegate, cannot bind
Only use scoreboard port.

PROTOCOL FORMAT: {RYESCR}{tag}value{tag}value...
Starts with {RYESCR} header, then tagged fields.

CONFIRMED TAGS:
  {fr}  = ring number
  {1}   = entry number
  {2}   = horse name
  {3}   = rider name
  {4}   = owner
  {5}   = separator (empty)
  {8}   = rank (FINISH SIGNAL — appears with final placing)
  {13}  = TA: time allowed (strip "TA: " prefix)
  {14}  = JUMP faults (strip "JUMP " prefix)
  {15}  = TIME faults (strip "TIME " prefix)
  {17}  = elapsed seconds (whole number while running, decimal at finish)
  {18}  = TTB: Time To Beat (format: "0 / 6.670" — target/previous round time)
          Note: TTB disappears mid-round, unreliable
  {23}  = countdown (negative e.g. "-44" = 44 seconds remaining in CD)

PHASE DETECTION (from which tags are present):
  INTRO    = entry/horse/rider present, no cd or elapsed, no rank
  CD       = {23} countdown present
  ONCOURSE = {17} elapsed present, no {8} rank
  FINISH   = {8} rank present

ONE PACKET PER SECOND during CD and ONCOURSE phases.

## UDP EVENTS — FIRED BY WATCHER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Watcher detects phase transitions and fires clean events into udpEvents[] queue.
Events include UTC timestamp for browser clock sync.

EVENTS:
  INTRO         Horse called — entry, horse, rider, ta
  CD_START      Countdown begins — + countdown, round, label
  CD_STOPPED    Countdown frozen 2.5s — + last countdown
  CD_RESUMED    Countdown moving again — + new countdown
  RIDE_START    Start beam triggered — + ta, jumpFaults, timeFaults, round, label
  FAULT         Jump faults changed — + elapsed, jumpFaults, timeFaults
  CLOCK_STOPPED Elapsed frozen 2.5s — + last elapsed
  CLOCK_RESUMED Elapsed moving again — + new elapsed
  FINISH        RANK tag appeared — + rank, jumpFaults, timeFaults, round, label

ROUND INFERENCE (in RIDE_START, CD_START, FINISH events):
  Priority 1: Match UDP TA to round config in .cls header
  Priority 2: If ambiguous (same TA multiple rounds) → check .cls entry state
              r1Time empty = Round 1, r2Time empty = Round 2, both = Round 3
  Priority 3: ScoringMethodCode=9 (two-phase) → R2/R3 blocks, labels "Phase 1"/"Phase 2"
  Labels: "Round 1", "Jump Off", "Round 3", "Phase 1", "Phase 2"

OPERATOR NOTE: Operator may skip INTRO — CD_START is the reliable round start signal.

## LIVE DISPLAY ARCHITECTURE — DECIDED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Instead of streaming every UDP packet (60-80 per horse), watcher fires 4-5 
clean events per horse. Website uses browser clock to tick locally.

CLOCK APPROACH:
- RIDE_START fires with UTC timestamp
- Browser starts ticking from 0 using local clock
- FAULT events update fault counter
- CLOCK_STOPPED/RESUMED pause/resume the local ticker
- FINISH fires → show "FINISHING..." → wait up to 5s for .cls update
- .cls update arrives → replace estimated time with official time
- Live display labeled "Times are approximate" (small, bottom of live box)

TIME DISPLAY RULES (CRITICAL):
- NEVER use minutes:seconds format — ALWAYS whole seconds with decimals
- Non-FEI: always 3 decimal places e.g. 82.620, 154.340
- FEI: 2 decimal places e.g. 82.62
- Live ticking clock shows whole seconds only (UDP sends whole seconds)
- Official time from .cls shows full precision

ON DECK LOGIC:
- Find current on-course entry from UDP entry number
- Look up their RideOrder in .cls
- On deck = lowest RideOrder among entries where hasGone=false AND entry ≠ current
- If RideOrder all 0 (not posted) → show pending entries in entry number order
- Edge case: out-of-order riders handled — just track lowest unscored RideOrder

TIME FAULTS (client-side calculation):
- Browser calculates time faults from ticking clock using .cls header data
- Formula: secondsOver = elapsed - TA; if secondsOver > 0: timeFaults = floor(secondsOver / timeInterval) × faultsPerInterval
- No need to wait for UDP {15} time faults tag
- Eliminates most of the time fault UDP monitoring
