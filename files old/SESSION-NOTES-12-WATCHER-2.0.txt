## DATA FLOW — HOW IT ALL CONNECTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SCORING COMPUTER → WATCHER → WORKER → KV → WEBSITE

1. Watcher reads .cls file change
   → Parses into clean JSON (see west_data.json schema below)
   → POSTs to Worker endpoint: POST /postClassData
   → Worker writes to KV: live:{slug}:{ring}  TTL: 2 hours
   → Worker writes to D1: permanent archival

2. Watcher detects UDP event (RIDE_START, FAULT, FINISH etc)
   → Drains udpEvents[] queue
   → POSTs to Worker endpoint: POST /postUdpEvent
   → Worker writes to KV: event:{slug}:{ring}  TTL: 5 minutes

3. Watcher sends heartbeat every 60 seconds
   → POST /heartbeat
   → Worker writes to KV: heartbeat:{slug}:{ring}
   → Allows remote monitoring of watcher status

4. Website polls KV every 2-3 seconds
   → GET /getLiveClass?slug={slug}&ring={ring}
   → Returns both live:{...} and event:{...} in one response
   → Website renders standings + live banner

5. Browser clock handles live ticking
   → RIDE_START event has UTC timestamp
   → Browser calculates elapsed = (now - rideStartTimestamp) / 1000
   → Browser calculates time faults from elapsed vs TA using header data
   → FAULT event updates fault counter
   → CLOCK_STOPPED pauses ticker, CLOCK_RESUMED resumes

KV KEYS:
  live:{slug}:{ring}       full class standings JSON, TTL 2hr
  event:{slug}:{ring}      latest UDP event, TTL 5min
  heartbeat:{slug}:{ring}  watcher alive signal, TTL 2min
  (existing production keys untouched)

## UDP PROTOCOL — CONFIRMED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Scoreboard port: config.dat col[1] (e.g. 29711)
Live data port 29215: PERMANENTLY LOCKED by Ryegate — do not attempt to bind

FORMAT: {RYESCR}{tag}value{tag}value...

TAGS:
  {fr}  ring number
  {1}   entry number
  {2}   horse name
  {3}   rider name
  {4}   owner
  {8}   rank — FINISH SIGNAL (strip "RANK " prefix)
  {13}  time allowed (strip "TA: " prefix)
  {14}  jump faults (strip "JUMP " prefix)
  {15}  time faults (strip "TIME " prefix)
  {17}  elapsed seconds (whole number mid-run, decimal at finish)
  {18}  TTB time to beat (format: "0 / 6.670" — unreliable, disappears mid-round)
  {23}  countdown (negative e.g. "-44")

PHASES:
  IDLE      no active horse
  INTRO     horse called, no CD or elapsed
  CD        {23} countdown present
  ONCOURSE  {17} elapsed present, no {8} rank
  FINISH    {8} rank present

EVENTS FIRED BY WATCHER:
  INTRO         horse called
  CD_START      countdown begins (+ round, label)
  CD_STOPPED    countdown frozen 2.5s
  CD_RESUMED    countdown moving again
  RIDE_START    start beam (+ round, label, UTC timestamp for browser clock)
  FAULT         jump faults changed
  CLOCK_STOPPED elapsed frozen 2.5s
  CLOCK_RESUMED elapsed moving again
  FINISH        RANK tag appeared (+ round, label)

ROUND INFERENCE (in CD_START, RIDE_START, FINISH):
  1. Match UDP TA to round config in .cls header rounds array
  2. If ambiguous → check .cls entry: r1Time empty=R1, r2Time empty=R2, both=R3
  3. ScoringMethod=9 two-phase → labels "Phase 1"/"Phase 2", uses R2/R3 blocks
  Labels: "Round 1", "Jump Off", "Round 3", "Phase 1", "Phase 2"
  INTRO is optional — operator may skip it. CD_START is reliable.

## CLS FILE FORMAT — KEY FACTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Full detail in CLS-FORMAT.md. Key facts only here:

ENTRY ROW LENGTHS:
  Hunter (H):  always 55 cols
  Farmtek (J): always 40 cols (no timestamp blocks)
  TIMY (T):    always 85 cols (includes 3 round timestamp blocks)
  Unformatted (U): 14 cols

FILE WRITE BEHAVIOR:
  Ryegate writes ENTIRE file atomically at end of each round
  File does NOT update while horse is on course
  On-course state = UDP only
  Atomic write → no partial reads needed

JUMPER ENTRY CONFIRMED COLS:
  15: R1Time       elapsed seconds e.g. 36.360
  16: R1JumpFaults fault points e.g. 4, 8
  17: R1Total      R1Time + R1JumpFaults
  22: R2Time       JO elapsed seconds
  24: R2Total      JO total
  25: R2JumpFaults
  35: StatusCode   RF=RiderFall, EL=Eliminated, WD=Withdrawn etc
  36: R1_HasGone_or_CDStart  (Farmtek=1 competed | TIMY=CDStart TOD)
  44: R1_RideStart TOD  ← CONFIRMED
  51: R1_RideEnd TOD    ← CONFIRMED
  52: R2_CDStart TOD    ← CONFIRMED
  59: R2_RideStart TOD  ← CONFIRMED
  66: R2_RideEnd TOD    ← CONFIRMED

HUNTER ENTRY CONFIRMED COLS:
  13: GoOrder
  14: CurrentPlace (live standing)
  15: R1Score (standard single-judge classes)
  24: R2Score (two-round classics)
  42: R1Total
  43: R2Total (classics/derbies)
  45: CombinedTotal
  49: HasGone_R1 = 1  ← NOT col 40 (col 40 is always 0)
  50: HasGone_R2 = 1  (two-round classes only)
  52: StatusCode (EX=Excused, RF=RiderFall, OC=OffCourse)

HUNTER SCORE DETECTION:
  Has competed: col[49]=='1' OR col[50]=='1'
  Standard: col[49]=='1' AND col[15] non-zero
  Classic/Derby: col[50]=='1' (all rounds complete)

JUMPER SCORE DETECTION:
  R1 complete: col[15] OR col[17] OR col[14] non-zero
  R2 complete: col[22] OR col[24] non-zero

TIME FAULTS (client-side formula):
  secondsOver = elapsed - TA
  if secondsOver > 0:
    timeFaults = floor(secondsOver / timeInterval) × faultsPerInterval
  Standard class: TA=78, FPI=1, TI=1 → 1 fault per second over
