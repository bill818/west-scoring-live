5. HUNTER UDP ARCHITECTURE DECISION — FINAL
   The .cls file does NOT write when a horse goes on course.
   The .cls file DOES write immediately when a score is posted.
   Therefore hunter UDP is used for ONE PURPOSE ONLY: ON COURSE signal.

   THE PIPELINE:
   {fr}=11 INTRO fires → horse is ON COURSE → write to KV
   .cls file changes   → horse DONE, score posted → clear KV, write D1

   NO other hunter UDP parsing needed. .cls is authoritative for all scores.
   FINISH UDP packets ({fr}=12, {fr}=16) are logged but not acted on.

6. NEW HUNTER UDP TAGS CONFIRMED — 2026-03-29
   Frame 11 cycles between two page layouts:
   Page A: {1}entry {2}horse {3}rider {4}owner {17}=SB message (NOT elapsed)
   Page B: {1}entry {2}horse {18}sire {19}="X" (breeding filler) {20}dam

   {fr}=16 confirmed = DERBY FINISH (not standard O/F)
